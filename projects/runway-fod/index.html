Runway FOD (Foreign Object Debris) Detection

Detection stack that pairs YOLO with RCNN persistence tracking so airside teams can triage debris before it becomes costly FOD.

0% Loading status…

Problem statement

Detect objects in field of view and classify them as either cleared (in view and tagged as not a threat to operations) or not cleared (in view and tagged as a threat to operations). The system prioritizes detections based on size, material, and proximity to active runways.

Approach

  • Collect diverse runway imagery (day/night, wet/dry, various camera angles) and annotate bounding boxes and threat labels.
  • Train an object detection model (e.g., YOLOv8 / Faster R-CNN) with threat-class labels and confidence calibration.
  • Post-processing pipeline to fuse detections across frames, estimate object persistence, and reduce false positives.
  • Operator UX: prioritize alerts and provide explainable metadata (size, distance estimate, sample frame) to accelerate clearing operations.

Implementation notes

  • Edge deployment option with model quantization for onsite inference on near-runway cameras.
  • Cloud analytics can be used for model updates and reporting; operational details are out of scope for this public summary.
  • APIs for integrating with airport operations dashboards and maintenance ticketing systems (integration specifics are implementation-dependent).
