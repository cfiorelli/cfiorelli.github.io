Mini-LLM: Semantic Scholar Assistant

Lightweight assistant that pairs conversational search with auto-generated API snippets for Semantic Scholar developers.

0% Loading status…

Goals

  • Provide a chat interface to search and summarize research results from Semantic Scholar.
  • Generate example API calls and code snippets to fetch paper metadata or PDFs.
  • Allow users to ask for citation formatting and quick literature overviews.

Implementation ideas

  • Frontend: small React/Vue/vanilla app with streaming chat UI.
  • Backend: proxy to Semantic Scholar API + lightweight LLM to rephrase and summarize results.
  • Security: rate-limiting, caching, and provenance tracking for generated code snippets.

Example snippets

Small runnable examples to get started.

Python (requests) — fetch paper metadata 
import requests
resp = requests.get("https://api.semanticscholar.org/graph/v1/paper/ARXIV:2301.10140?fields=title,abstract,authors,year")
print(resp.json())
curl — search 
curl "https://api.semanticscholar.org/graph/v1/paper/search?query=semantic+search&limit=5"
Client-side pattern (proxy) 
// Client calls your backend: GET /api/s2/paper/ARXIV:2301.10140
// Backend forwards to Semantic Scholar with your API key and returns the response.
fetch('/api/s2/paper/ARXIV:2301.10140').then(r=>r.json()).then(console.log);

Backend notes

In production you should keep API keys server-side and use a small proxy to forward requests; this helps protect credentials and apply caching/rate limits. Implementation details are intentionally omitted from the public site.
